🚀 MINI PROJECT: SMART E-COMMERCE CHECKOUT SYSTEM
📝 Ngữ cảnh (Context)
Bạn cần xây dựng một hệ thống giả lập việc quản lý giỏ hàng và thanh toán cho một trang thương mại điện tử. Hệ thống này phải xử lý được các mặt hàng khác nhau, tính toán chiết khấu và mô phỏng việc gọi API thanh toán một cách bất đồng bộ.

🛠 Yêu cầu kỹ thuật (Requirements)
1. Xây dựng cấu trúc Class (OOP)
Class Product (Base Class):
Thuộc tính: name, price.
Phương thức: getDetails() (In ra tên và giá).
Class Electronics (Kế thừa Product):
Thuộc tính riêng: warranty (số tháng bảo hành).
Đa hình: Ghi đè hàm getDetails() để in thêm thông tin bảo hành.
Class Clothing (Kế thừa Product):
Thuộc tính riêng: size.
Đa hình: Ghi đè hàm getDetails() để in thêm kích cỡ.
2. Xây dựng hệ thống Giỏ hàng (Encapsulation)
Class ShoppingCart:
Thuộc tính #items (Mảng Private - không cho phép sửa trực tiếp từ ngoài).
Phương thức addItem(product): Thêm sản phẩm vào mảng.
Phương thức showCart(): In ra danh sách sản phẩm hiện có.
Phương thức calculateTotal(): Tính tổng tiền.
3. Xử lý Thanh toán (Asynchronous)
Tạo một hàm processPayment(totalAmount) trả về một Promise.
Nếu totalAmount > 0: Resolve sau 2 giây với thông báo "Thanh toán thành công".
Nếu totalAmount <= 0: Reject với thông báo "Giỏ hàng trống, không thể thanh toán".
