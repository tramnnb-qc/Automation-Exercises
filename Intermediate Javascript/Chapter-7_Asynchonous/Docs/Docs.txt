Chapter 7: ASYNCHRONOUS
1. Bản chất của JavaScript: Đơn luồng (Single-threaded)
Hãy tưởng tượng "JavaScript giống như một quán ăn chỉ có duy nhất 1 đầu bếp".
Ông ấy không thể làm hai việc cùng một lúc (ví dụ: vừa chiên trứng vừa thái thịt bằng 1 đôi tay).
Ông ấy thực hiện các yêu cầu theo thứ tự: việc nào đến trước làm trước (Synchronous - Đồng bộ).
Vấn đề: Nếu một khách hàng gọi món "Bò hầm" mất 30 phút, chẳng lẽ đầu bếp phải đứng nhìn nồi thịt hầm rồi bắt tất cả khách hàng phía sau đợi 30 phút mới được gọi món? => Đây gọi là Blocking (Làm nghẽn hệ thống).

2. Cơ chế xử lý "Thông minh": Event Loop
Để không bị nghẽn, JavaScript sử dụng cơ chế Bất đồng bộ.
Call Stack (Bàn làm việc): Nơi các lệnh được thực hiện ngay lập tức.
Web APIs / Background Tasks (Nhà kho/Lò nướng): Nơi gửi các tác vụ tốn thời gian (hẹn giờ, gọi API, đợi Element).
Callback Queue (Hàng đợi): Nơi các món ăn đã nấu xong chờ được bưng ra.
Event Loop (Người phục vụ): Kiểm tra nếu "Bàn làm việc" trống, ông ấy sẽ bưng món từ "Hàng đợi" ra trả cho khách.



3. Callback - Cách tiếp cận cổ điển
Khái niệm: Bạn đưa một hàm (Callback) cho một hàm khác và dặn: "Khi nào xong việc thì gọi cái hàm này nhé".
Ví dụ thực tế: Bạn đi trà sữa, nhân viên đưa cho bạn cái thẻ rung. Khi nào nước xong, thẻ rung lên (Callback), bạn ra lấy nước.
Ex:
function step1(callback) {
    setTimeout(() => {
        console.log("1. Mở trình duyệt");
        callback();
    }, 1000);
}

step1(() => {
    console.log("2. Truy cập URL");
});


Vấn đề - "Callback Hell": Khi bạn có quá nhiều bước liên tiếp (Đăng ký -> Nhập username -> Check Data -> nhập password -> Check Data -> Nhập email -> Check Data), code sẽ bị thụt lề vào trong như một hình tam giác (Pyramid of Dom), cực kỳ khó đọc và bảo trì.


4. Promise - Lời hứa (ES6)
Để giải quyết "địa ngục" trên, Promise ra đời. Nó thay thế việc lồng các hàm vào nhau bằng một chuỗi các bước (.then).
3 Trạng thái của một Promise:
Pending: Đang chờ xử lý (Đang pha cà phê).
Fulfilled: Thành công (Đã có cà phê). -> Chạy vào .then()
Rejected: Thất bại (Hết hạt cà phê). -> Chạy vào .catch()
Code ví dụ:
const openBrowser = new Promise((resolve, reject) => {
    let isSuccess = true;
    if (isSuccess) resolve("Mở trình duyệt thành công!");
    else reject("Lỗi khởi động!");
});

openBrowser
    .then((msg) => {
        console.log(msg);
        return "Truy cập URL"; // Tiếp tục bước tiếp theo
    })
    .then((step) => console.log(step))
    .catch((err) => console.error(err));


Ưu điểm: Code trông "thẳng" hơn, dễ quản lý lỗi tập trung ở một chỗ (.catch).
Nhược điểm:
Ví dụ ta có các hàm dưới:
login() -> Trả về token. //JWT
getCart(token) -> Trả về cartId.
addItem(cartId, productId) -> Trả về status.
checkout(status) -> Trả về orderId





// 1. Hàm Đăng nhập
const login = (user, pass) => {
    return new Promise((resolve, reject) => {
        console.log(`--- 1. Đang đăng nhập cho user: ${user} ---`);
        setTimeout(() => {
            if (user === "admin" && pass === "123") {
                resolve("TOKEN_SECRET_123"); // Trả về token
            } else {
                reject("Sai tài khoản hoặc mật khẩu!");
            }
        }, 1500); // Đợi 1.5s
    });
};

// 2. Hàm lấy Giỏ hàng
const getCart = (token) => {
    return new Promise((resolve, reject) => {
        console.log(`--- 2. Đang lấy giỏ hàng với token: ${token} ---`);
        setTimeout(() => {
            if (token) {
                resolve("CART_ID_999"); // Trả về ID giỏ hàng
            } else {
                reject("Token không hợp lệ!");
            }
        }, 1000);
    });
};

// 3. Hàm thêm Sản phẩm
const addItem = (cartId, productId) => {
    return new Promise((resolve, reject) => {
        console.log(`--- 3. Đang thêm sản phẩm ${productId} vào giỏ ${cartId} ---`);
        setTimeout(() => {
            resolve("SUCCESS_STATUS"); // Trả về trạng thái thành công
        }, 1000);
    });
};

// 4. Hàm Thanh toán
const checkout = (status) => {
    return new Promise((resolve, reject) => {
        console.log("--- 4. Đang xử lý thanh toán... ---");
        setTimeout(() => {
            if (status === "SUCCESS_STATUS") {
                resolve("ORDER_ID_CONFIRMED_001"); // Trả về mã đơn hàng
            } else {
                reject("Thanh toán thất bại!");
            }
        }, 2000);
    });
};

5. Asynchronous - Xử lý "Đợi chờ"
Đây là phần quan trọng nhất. Nếu không hiểu cái này, học viên sẽ không bao giờ làm được Automation Test ổn định.
Khái niệm : Tại sao JS lại chạy "nhảy cóc"?
Ví dụ: Gửi yêu cầu lên Server -> Server chưa trả về ngay -> JS chạy tiếp dòng dưới -> Lỗi vì chưa có dữ liệu.
Async/Await: Cách biến code "bất đồng bộ" trông như "đồng bộ" (chạy từ trên xuống dưới).


async function performTest() {
    console.log("1. Mở trình duyệt");
    await delay(2000); // Giả lập đợi 2s
    console.log("2. Click nút Login");
}

Giải quyết cho bài toán purchaseFlow
async function purchaseFlow() {
 try {
   const token = await login("admin", "123");
   const cartId = await getCart(token);
   const status = await addItem(cartId, "PROD_001");
   const orderId = await checkout(status);

   console.log(`Thành công! Đơn hàng: ${orderId} với token: ${token}`);
 } catch (error) {
   console.error("Script thất bại tại bước nào đó:", error.message);
 }
}

7. Xử lý lỗi với Try...Catch
Giúp script không bị "chết" giữa chừng và biết cách báo cáo lỗi chuyên nghiệp.
Cấu trúc Try...Catch...Finally:
try: Thử chạy code nguy hiểm (ví dụ: tìm một Element có thể không tồn tại).
catch: Chộp lấy lỗi và xử lý (ví dụ: in lỗi ra console đỏ chót).
finally: Dọn dẹp (ví dụ: đóng trình duyệt bất kể kết quả ra sao).
Thực hành: Viết một hàm chia 2 số, nếu số chia là 0 thì dùng throw new Error() để đẩy lỗi vào catch.


Bài tập:
Hãy yêu cầu học viên viết lại đoạn kịch bản test sau từ Callback sang Promise:
wait(1s) -> In ra "Start Test".
wait(1s) -> In ra "Doing Test".
wait(1s) -> In ra "End Test".
Bài tập 1: "Làm phẳng" kim tự tháp (Refactoring)
Đề bài: Cho một đoạn code sử dụng Callback cực kỳ rắc rối dưới đây. Đoạn code này mô phỏng việc Mở trình duyệt -> Nhập URL -> Tìm Element -> Click.
function openBrowser(callback) {
    setTimeout(() => {
        console.log("1. Đã mở trình duyệt");
        callback();
    }, 1000);
}

function enterUrl(url, callback) {
    setTimeout(() => {
        console.log("2. Đã nhập URL: " + url);
        callback();
    }, 1000);
}

function clickButton(callback) {
    setTimeout(() => {
        console.log("3. Đã click nút Login");
        callback();
    }, 1000);
}





// Thực thi (Địa ngục Callback)
openBrowser(() => {
    enterUrl("https://test.com", () => {
        clickButton(() => {
            console.log("✅ Test hoàn tất!");
        });
    });
});


Yêu cầu: 
1. Viết lại 3 hàm trên dưới dạng Promise. 
2. Sử dụng Async/Await để thực thi chuỗi hành động trên sao cho code trông "thẳng" và dễ đọc nhất.

Bài tập 2: "Bẫy lỗi" hệ thống (Error Handling)
Đề bài: Trong Automation, đôi khi Server sẽ trả về lỗi (ví dụ: Timeout 404). Bạn cần viết một script "nồi đồng cối đá" không bị crash khi gặp lỗi.
Yêu cầu: 
1. Viết một hàm fetchDataFromServer() trả về một Promise. 
2. Sử dụng Math.random() bên trong hàm: * Nếu số ngẫu nhiên > 0.5: Resolve với nội dung "Dữ liệu đã sẵn sàng". * Nếu số ngẫu nhiên <= 0.5: Reject với nội dung "Lỗi kết nối Server!". 
3. Viết một hàm runTest() sử dụng Async/Await để gọi hàm trên. 
4. Quan trọng: Phải dùng try...catch để: * Nếu thành công: In ra dữ liệu. * Nếu thất bại: In ra thông báo lỗi màu đỏ và dòng chữ "Đang đóng trình duyệt để dọn dẹp...".

Bài tập 3: Kịch bản Test song song (Parallel vs Sequential)
Đề bài: Giả sử bạn cần kiểm tra 3 Page khác nhau: HomePage, ProductPage, và ContactPage. Mỗi trang mất 2 giây để load.
Yêu cầu:
Viết một hàm checkPage(pageName) trả về Promise và hoàn thành sau 2 giây.
Kịch bản A (Tuần tự): Dùng await để kiểm tra từng trang một (Trang 1 xong mới đến trang 2). Sau đó in ra tổng thời gian chạy (dự kiến là 6 giây).
Kịch bản B (Song song - Nâng cao): Tìm hiểu về Promise.all() để kích hoạt kiểm tra cả 3 trang cùng một lúc. Sau đó in ra tổng thời gian chạy (dự kiến chỉ mất 2 giây).
