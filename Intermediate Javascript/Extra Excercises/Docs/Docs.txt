Chapter 7: Asynchronous
🏆 BÀI TẬP: HỆ THỐNG ĐIỀU PHỐI TEST (SMART TEST RUNNER)
Ngữ cảnh: Bạn đang xây dựng một hệ thống chạy Automation Test có giới hạn về tài nguyên (Timeout). Bạn cần thực thi một danh sách các nhiệm vụ, nhưng mỗi nhiệm vụ có độ trễ và độ ưu tiên khác nhau.
Yêu cầu:
Tạo 3 hàm giả lập (Promises):
taskA(): Mất 3 giây để hoàn thành, in ra "Nhiệm vụ A xong".
taskB(): Mất 1 giây để hoàn thành, in ra "Nhiệm vụ B xong".
taskC(): Mất 2 giây để hoàn thành, in ra "Nhiệm vụ C xong".
Thực thi hỗn hợp:
Chạy taskA và taskB song song (Parallel).
Ngay sau khi cả A và B đều xong, hãy lấy kết quả của chúng để làm đầu vào cho taskC.
Toàn bộ quá trình này phải nằm trong một khối try...catch.
Thử thách "Cực khó" (Race Condition & Timeout):
Sử dụng Promise.race() để đặt một "bom hẹn giờ" (Timeout) là 3.5 giây.
Nếu tổng thời gian chạy của cả cụm (A+B rồi mới đến C) vượt quá 3.5 giây, hệ thống phải dừng lại và in ra lỗi: "Test bị hủy do quá thời gian quy định!".
Nếu hoàn thành kịp, in ra: "Tất cả Test Case đã Pass trong thời gian cho phép!".
Gợi ý tư duy: * Bạn sẽ cần kết hợp Promise.all() cho A và B.
Sau đó dùng kết quả của Promise.all() để gọi C.
Cuối cùng, bọc toàn bộ chuỗi đó vào một Promise.race() với một Promise "Timeout".

Question 6: What is the exact output order of the following code?
console.log('A');
setTimeout(() => console.log('B'), 0);
Promise.resolve().then(() => console.log('C'));

async function quickTask() {
    console.log('D');
    await Promise.resolve();
    console.log('E');
}

quickTask();

console.log('F');


Answer: B. A, D, F, C, E, B
Rationale: 1. Synchronous code executes first: A, then enters quickTask() to print D, then continues after the function call to print F. 2. Microtasks (Promises) are next: C (from the standalone Promise) and E (the code after await in the async function). 3. Macrotasks (setTimeout) run last: B.

Question 7: In an Automation framework (like Playwright or WDIO), what is the primary danger of using Array.prototype.map() with an async callback without wrapping it in Promise.all()?
Answer: B. The map function will return an array of pending Promises.
Rationale: map executes the callback for each item and moves on immediately. Since the callback is async, it returns a Promise. Without await Promise.all(...), the resulting variable is just a list of "Promises" that haven't finished yet, leading to "undefined" or errors when you try to use the data.

Question 8: How does await behave inside a try...catch block if the awaited Promise is rejected?
Answer: C. The await expression throws the rejected value as an exception.
Rationale: This is the "magic" of async/await. It allows us to use standard try...catch blocks to handle asynchronous errors as if they were normal synchronous errors, making the code much cleaner than using .catch() chains.
